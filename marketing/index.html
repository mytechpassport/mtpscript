The Problem With AI + Existing Languages

AI can write code — but existing backend languages trust the code too much.

In JavaScript, TypeScript, Go, Rust, and Python:

  • Authority is ambient
  • State is shared
  • Effects are implicit
  • Runtime behavior is nondeterministic
  • Safety depends on developer discipline
The Risk to Your System

AI-generated code can accidentally:

  • Access I/O it shouldn’t
  • Leak state across requests
  • Behave differently under load
  • Pass tests but fail audits

The problem is not AI.
The problem is the language.

MTPScript Is Designed for AI-Assisted Coding

AI-Safe by Construction

MTPScript is intentionally strict so that AI cannot generate unsafe code. The language removes entire categories of mistakes — not through linting, but through hard semantic rules.

What MTPScript Is

JavaScript Syntax. Regulated & AI-Safe Semantics.

MTPScript is a JavaScript-shaped language that compiles to a strictly constrained JavaScript subset, executed by a sealed MicroQuickJS runtime.

If you know JavaScript or TypeScript, you already know the syntax. What changes is what the language allows — and forbids.

JavaScript vs MTPScript

Same Intent. Different Guarantees.

JavaScript / TypeScript
Implicit I/O 
app.post("/users", async (req, res) => {
  // Implicit: DB access, network,
  // async scheduling, shared state
  const user = await db.insert(req.body);
  console.log("created user", user);
  res.json(user);
});
MTPScript
Explicit Effects 
api POST /users
uses { db, log } {
  // Explicit capabilities only
  let user = db.insert(req.body)
  log.info("created user", user)
  respond json(user)
}

Per-Request Isolation Without Cold Starts

Each request runs in its own isolated VM, created via a snapshot-clone model:

  1. 1 Immutable build-time snapshot
  2. 2 Copy-on-write VM clone
  3. 3 Execute exactly one request
  4. 4 Discard VM immediately

Why MTPScript Exists

Most languages assume:

  • Humans write code carefully
  • Runtime issues are caught later
  • Safety is enforced by convention

MTPScript assumes:

  • Code will be generated by AI
  • Execution must be provably safe
  • Mistakes must be impossible, not discouraged

MTPScript is a language you can trust — even when the code wasn’t written by you.
